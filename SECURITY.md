# Security Policy

## Reporting security concerns

Please do not disclose suspected vulnerabilities publicly in issues, discussions, pull requests, blog posts, or social media.

Report security concerns privately via the canonical request path:

[Request access](https://ai-admissibility.com/request)

Include:

- repository name;
- affected repository surface;
- reproducible steps;
- expected vs actual behavior;
- impact summary.

## Safe disclosure

Do not publish secrets, credentials, private keys, internal endpoints, server details, deployment mechanics, or other sensitive implementation details in public reports.

## Response

Reports are reviewed privately. Validation and response timing depend on reproducibility, scope, and severity.
